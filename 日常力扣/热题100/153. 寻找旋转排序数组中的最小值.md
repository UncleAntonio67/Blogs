🏷中等 #二分查找 #数组 

已知一个长度为 `n` 的数组，预先按照升序排列，经由 `1` 到 `n` 次 **旋转** 后，得到输入数组。例如，原数组 `nums = [0,1,2,4,5,6,7]` 在变化后可能得到：

- 若旋转 `4` 次，则可以得到 `[4,5,6,7,0,1,2]`
- 若旋转 `7` 次，则可以得到 `[0,1,2,4,5,6,7]`

注意，数组 `[a[0], a[1], a[2], ..., a[n-1]]` **旋转一次** 的结果为数组 `[a[n-1], a[0], a[1], a[2], ..., a[n-2]]` 。

给你一个元素值 **互不相同** 的数组 `nums` ，它原来是一个升序排列的数组，并按上述情形进行了多次旋转。请你找出并返回数组中的 **最小元素** 。

你必须设计一个时间复杂度为 `O(log n)` 的算法解决此问题。

示例 1：

>输入：`nums = [3,4,5,1,2]`
>输出：`1`
>解释：`原数组为 [1,2,3,4,5] ，旋转 3 次得到输入数组。`

示例 2：

>输入：`nums = [4,5,6,7,0,1,2]`
>输出：`0`
>解释：`原数组为 [0,1,2,4,5,6,7] ，旋转 4 次得到输入数组。`

示例 3：

>输入：`nums = [11,13,15,17]`
>输出：`11`
>解释：`原数组为 [11,13,15,17] ，旋转 4 次得到输入数组。`

**提示：**

- `n == nums.length`
- `1 <= n <= 5000`
- `-5000 <= nums[i] <= 5000`
- `nums` 中的所有整数 **互不相同**
- `nums` 原来是一个升序排序的数组，并进行了 `1` 至 `n` 次旋转


## 解题思路

- 如果 `nums[mid] > nums[right]`：
    - 说明最小值在 **右半部分**（排除了左半部分），
    - 因此 `left = mid + 1`
- 否则（`nums[mid] <= nums[right]`）：
    - 说明最小值在 **左半部分**（可能是 mid 本身），

    - 因此 `right = mid
循环结束后，`left == right`，就是最小值：

| **维度**    | **分析内容**                                                                                                                                                                                                                         |
| --------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **核心思路**  | **利用右边界收缩区间**。<br>1. **比较基准**：将 `mid` 与 `right` 进行比较，因为 `right` 始终位于旋转后的“第二段”有序序列中。<br>2. **左移逻辑**：如果 `nums[mid] > nums[right]`，说明最小值一定在 `mid` 的右侧，故 `left = mid + 1`。<br>3. **右移逻辑**：否则，`mid` 可能就是最小值，或者最小值在左侧，故 `right = mid`。 |
| **循环条件**  | **`left < right`**。<br>当 `left == right` 时，区间缩减为一个点，该点即为最小值，循环自然终止。                                                                                                                                                              |
| **时间复杂度** | $O(\log N)$<br>标准的二分查找效率。                                                                                                                                                                                                        |
| **空间复杂度** | $O(1)$<br>空间占用极低。                                                                                                                                                                                                                |
## 方法一：

```python
class Solution:
    def findMin(self, nums: List[int]) -> int:
        left, right = 0, len(nums) - 1
        
        while left < right:
            mid = (left + right) // 2 
            if nums[mid] > nums[right]:
                left = mid + 1
            else:
                right = mid
        return nums[left]
```
