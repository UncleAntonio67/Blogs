🏷中等 #数组 #回溯 

给你一个 **无重复元素** 的整数数组 `candidates` 和一个目标整数 `target` ，找出 `candidates` 中可以使数字和为目标数 `target` 的 所有 **不同组合** ，并以列表形式返回。你可以按 **任意顺序** 返回这些组合。

`candidates` 中的 **同一个** 数字可以 **无限制重复被选取** 。如果至少一个数字的被选数量不同，则两种组合是不同的。 

对于给定的输入，保证和为 `target` 的不同组合数少于 `150` 个。

**示例 1：**

>输入：`candidates = [2,3,6,7], target = 7`
>输出：`[[2,2,3],[7]]`
>**解释：**
>2 和 3 可以形成一组候选，2 + 2 + 3 = 7 。注意 2 可以使用多次。
>7 也是一个候选， 7 = 7 。仅有这两种组合。

**示例 2：**

>输入: `candidates = [2,3,5], target = 8`
>输出: `[[2,2,2,2],[2,3,3],[3,5]]`

**示例 3：**

>**输入:** `candidates = [2], target = 1`
>**输出:** `[]`

**提示：**

- `1 <= candidates.length <= 30`
- `2 <= candidates[i] <= 40`
- `candidates` 的所有元素 **互不相同**
- `1 <= target <= 40`


# 解题思路

 🎯 重点总结（你必须记住）

 ✔ 成功的路径：target 必须变成 **0**
 ✔ 无效的路径：target 变成负数（剪枝）
 ✔ 每次递归都用新的 target（减去当前选的数字）
 ✔ 有 start 才能保证不会出现无限循环或重复组合

组合类题目必须要 start，因为组合是无序的，不能出现倒退和重复。  
排列类题目不需要 start，因为排列是有序的，每种顺序都是不同答案。

| **维度**    | **分析内容**                                                                                                                                                                                          |
| --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **核心思路**  | **回溯法（完全背包变体）**。<br>1. **去重机制**：使用 `start` 索引确保搜索始终向后或停在原地，避免出现 `[2,3,3]` 和 `[3,2,3]` 这种重复组合。<br>2. **重复选取**：在递归调用时传递 `backtrack(i, ...)`，允许当前元素在下一层继续被选用。<br>3. **剪枝条件**：当 `target < 0` 时主动停止搜索。 |
| **空间复杂度** | $O(T/M)$<br>其中 $T$ 是目标值，$M$ 是所有候选数字中最小的一个。这是递归栈的最大深度（即由最小数字组成的组合长度）。                                                                                                                              |
| **时间复杂度** | $O(N^{\frac{T}{M}+1})$<br>这是一个指数级的复杂度。$N$ 为候选数字数量。虽然实际搜索空间会因为 `target` 的减小而迅速收缩，但理论上限较高。                                                                                                          |
| **优点**    | 1. **代码简洁**：利用递归栈自动处理所有可能的组合路径。<br>2. **内存优化**：通过 `path` 原地操作避免了大量中间列表的创建。                                                                                                                        |


## 方法一：

```python
class Solution:
    def combinationSum(self, candidates: List[int], target: int) -> List[List[int]]:
        res = []
        path = []

        def backtrack(start, target):
            if target == 0:
                res.append(path[:])
                return
            if target < 0:
                return
            for i in range(start, len(candidates)):
                path.append(candidates[i])
                backtrack(i, target - candidates[i])
                path.pop()

        backtrack(0, target)
        return res
```
