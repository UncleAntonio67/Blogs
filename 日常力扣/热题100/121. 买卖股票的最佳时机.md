🏷简单 #数组 #动态规划 

给定一个数组 `prices` ，它的第 `i` 个元素 `prices[i]` 表示一支给定股票第 `i` 天的价格。

你只能选择 **某一天** 买入这只股票，并选择在 **未来的某一个不同的日子** 卖出该股票。设计一个算法来计算你所能获取的最大利润。

返回你可以从这笔交易中获取的最大利润。如果你不能获取任何利润，返回 `0` 。

示例 1：

>输入：`[7,1,5,3,6,4]`
>输出：`5`
>解释：`在第 2 天（股票价格 = 1）的时候买入，在第 5 天（股票价格 = 6）的时候卖出，最大利润 = 6-1 = 5 。注意利润不能是 7-1 = 6, 因为卖出价格需要大于买入价格；同时，你不能在买入前卖出股票。`

示例 2：

>输入：`prices = [7,6,4,3,1]`
>输出：`0`
>解释：`在这种情况下, 没有交易完成, 所以最大利润为 0。`

**提示：**

- `1 <= prices.length <= 105`
- `0 <= prices[i] <= 104`

# 解题思路与实现

- 如果第i天卖出股票，则最大利润为(该天的股价-前面天数中最小的股价)，然后与已知的最大利润比较，如果大于则更新当前最大利润的值。

| **维度**    | **分析内容**                                                                                                                           |
| --------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| **核心思路**  | **动态维护最小值与最大差值**。<br>1. **记录低点**：用 `min_price` 捕捉到目前为止遇到的最便宜的买入机会。<br>2. **尝试卖出**：假设在 `min_price` 买入，在当前价格 `p` 卖出，更新 `max_profit`。 |
| **时间复杂度** | $O(N)$<br>只需要对价格数组进行一次扫描。                                                                                                          |
| **空间复杂度** | $O(1)$<br>仅使用了两个辅助变量。                                                                                                              |
| **适用限制**  | **只能交易一次**。如果你想在同一天买入并卖出是不产生利润的，且必须先买后卖。                                                                                           |
## 方法一：

```python
class Solution:
    def maxProfit(self, prices: List[int]) -> int:
        min_price = float('inf')
        max_profit = 0

        for p in prices:
            min_price = min(min_price,p)
            max_profit = max(max_profit, p-min_price)
        return max_profit
```
